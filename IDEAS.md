## Автопродление подписки (рекуррент Т-Банка)
- В Init передавать `Recurrent: "Y"` + `CustomerKey: tg_id` → при оплате Т-Банк токенизирует карту.
- Из webhook'а на CONFIRMED сохранять `RebillId` в БД.
- Background task в боте за 2 дня до истечения → `Charge(rebill_id, amount, order_id)` → списание без юзера.
- UI: чекбокс «Автопродление» на `/pay`, в `/profile` показывать состояние + кнопка отключить.
- Уведомление за 3 дня «Через 3 дня спишется 250 ₽, отменить» → ссылка-отключение.
- Обработка неудачного rebill: уведомление юзеру + отключение autopay + фолбэк на ручную оплату.
- Чеки ФЗ-54 Т-Банк формирует автоматически по тому же Receipt-шаблону.
- **Когда делать:** после 10-20 реальных оплат, чтобы понять retention. Опасно запускать без тестов.

## Per-device VPN (фаза 2, отложено)
- Каждое устройство = отдельный Marzban-юзер (свой VLESS UUID, свой sub_url).
- Таблица `user_devices(tg_id, mz_username, device_label, platform, added_at)`.
- Лимит устройств per plan: free/trial=1, paid=3-5.
- В боте: список устройств + "➕ Добавить устройство" + revoke каждого.
- Даёт: revoke украденной ссылки без касания остальных, честный учёт трафика per device,
  monetization lever (апгрейд плана не за срок, а за кол-во устройств).
- Модель рынка: так делают Nord/Express/Surfshark ("до 6 устройств на аккаунт").
