# Скрины для /guide — что снять

Страница `/guide` (templates/guide.html) готова принимать скрины: в каждом шаге
есть слот `<img src="/static/guide/<имя>.jpg" onerror="this.remove()">`. Пока
файла нет — слот сам исчезает (страница как сейчас). Кладём JPG/PNG (портрет
телефона/окно) в `landing/static/guide/` под точным именем — скрин появляется
автоматически, без правок кода. Идеально: чистый системный скриншот (не фото
экрана), ширина ≥ 600px.

Имена = `<платформа>-<номер шага>.{jpg|png}`.

## iOS (Karing) — `ios-1`…`ios-6`
- `ios-1` — экран «Your Privacy Comes First» (кнопка **Accept & Continue**)
- `ios-2` — начальная настройка: выбор **Языка (Русский)** / **Страны (RU)**
- `ios-3` — **«Добавление подписки»**: поле «Ссылка на подписку/содержание» + галочка ✓ вверху
- `ios-4` — iOS-диалог «Karing запрашивает разрешение на добавление конфигураций VPN» → **Разрешить**
- `ios-5` — главный экран, кнопка **«Соединить»** (или статус «Подключено»)
- `ios-6` — главный экран после настройки (дашборд, «Автовыбор») — опционально

## Android (FLClash) — `android-1`…`android-4`
- `android-1` — первый запуск: диалог «Отказ от ответственности» / «Уведомление о сборе данных»
- `android-2` — вкладка **Профили** → «+ Добавить профиль» → **URL** / «Импорт из URL» с полем ссылки
- `android-3` — **Панель управления**: кнопка ▶ / системный запрос «FlClash пытается подключиться к сети VPN» → OK / зелёная ✅
- `android-4` — проверка: главный экран с IP и растущим трафиком (блок «Обнаруж…»)

## Windows (FLClash) — `win-1`…`win-3`
> На macOS основной клиент — **Karing из App Store** (тот же, что на iOS), шаги совпадают
> с `ios-*`. FLClash на маке остался запасным вариантом (свёрнутый блок в гайде).
- `win-1` — установка / первый запуск (окно)
- `win-2` — **+ → Import from URL** с полем ссылки
- `win-3` — подключено (**Connected**), режим «Правило»
